# 진동 베어링 이상탐지 — 작업 백로그 (TASKS)

실시간 진동 신호에서 **정상/이상**을 판단하는 비지도 이상탐지 시스템.
정상 데이터만 학습하고, 통계 경로(임펄스)와 디노이징 오토인코더 경로(미묘한 이상)를 병렬로 두어 융합 판정한다.

---

## 표기 안내

- `- [ ]` 미완료 / `- [x]` 완료 (GitHub에서 클릭 가능한 체크박스로 보임)
- `Tx.y` = 에픽 x의 y번째 티켓
- ⭐ = MVP(최소 동작 버전)에 꼭 필요한 티켓
- 각 티켓 아래 `완료:` = 이게 끝났다고 인정하는 조건(Definition of Done)

**추천 순서:** ⭐ 티켓만 먼저 일직선으로 끝내 end-to-end가 한 번 돌아가는 최소 버전을 만든 뒤, 나머지로 살을 붙인다.

---

## 설계 원칙 (시작 전에 못 박아둘 것)

- [ ] **데이터 누수 차단** — 정규화 통계는 정상 train에서만 fit 후 저장·재사용. 고장 데이터는 학습·임계값 설정에 절대 사용 금지(평가 전용).
- [x] **윈도우 설계 기록** — 샘플링 주파수 / 윈도우 길이(CWRU 1회전 ≈ 1,600포인트) / 겹침 비율을 숫자로 확정해 config에 명시.
- [x] **노이즈-임펄스 감도 균형** — 디노이징 노이즈 강도를 튜닝 가능한 설정값으로 분리. 너무 세면 임펄스 감도가 죽음.
- [ ] **임계값 규칙** — 정상 오차 분포의 상위 백분위(예: 99%)나 평균+3σ로 설정. 이상 데이터를 보기 전에 정한다.
- [ ] **오탐 억제** — 한 윈도우가 아니라 연속 N개 윈도우가 이상일 때만 알람(디바운스).
- [x] **도메인 가정 명시** — 다루는 운전 조건 범위를 한정해 적는다(예: "1,750rpm/2hp"). 범위 밖은 향후 과제.
- [ ] **재현성** — 시드 고정, config 기반 파이프라인, 모델과 전처리 통계를 함께 저장.
- [ ] **평가 분리** — 지표는 AUC·pAUC + 정상 오탐률(FPR)로 고정. 고장 데이터는 평가 단계에서만 등장.

## EPIC 7 — 이상 유형 분류 (경로 C, 지도학습)

> A·B(비지도)가 "이상"으로 판정한 신호만 받아 "어떤 유형인지" 분류한다.
> 핵심 규칙: ① C는 A·B 융합 판정이 이상일 때만 호출(게이트). ② C는 CWRU에 있는 알려진 유형만 분류 가능 → 출력은 확정 진단이 아닌 참고 추정. ③ 전처리(윈도잉·스펙트로그램)는 A·B와 공유하되, 학습은 별도 모델로 따로.

- [ ] **T7.1 CWRU 이상 데이터 로드·클래스 정의**
  - 외륜·내륜·볼 등 결함 유형을 클래스로 정의 (이상 데이터만 사용)
  - 완료: 유형별 라벨·샘플 개수 출력
- [ ] **T7.2 라벨 분포 점검·불균형 처리**
  - 클래스 불균형 확인 후 가중치/층화 분할 적용
  - 완료: 클래스 분포표 + 층화된 train/val 분할
- [ ] **T7.3 CNN 분류기 학습**
  - 스펙트로그램 입력 CNN 지도학습, 시드 고정
  - (선택) B의 인코더를 특징 추출기로 재사용 → T6.3과 연결
  - 완료: 검증 정확도와 혼동행렬 출력
- [ ] **T7.4 게이트 연동**
  - A·B 융합 판정이 이상일 때만 C 호출하도록 파이프라인 연결
  - 완료: 정상 신호에는 C가 호출되지 않음을 확인
- [ ] **T7.5 분류 평가·한계 기록**
  - 정확도·혼동행렬(어떤 유형을 무엇으로 헷갈리는지)
  - 완료: 지표 리포트 + "알려진 유형만 분류 가능" 한계 명시
